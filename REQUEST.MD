# PCMonitor4DLG 需求文档

## 1. 项目概述

**PCMonitor4DLG** 是一个基于 Kotlin Multiplatform (Compose for Desktop) 的 PC 性能监控工具，主要功能是实时采集主机硬件状态数据，并通过蓝牙同步至 DA14585 蓝牙时钟设备（DLG 系列）进行显示。

**技术栈：**
- Kotlin Multiplatform + Compose Multiplatform (Desktop JVM)
- 蓝牙通信：BLE (Bluetooth Low Energy)

---

## 2. 功能需求

### 2.1 系统监控（顶部区域）

界面顶部区域实时展示以下主机状态信息，数据刷新间隔 **1 秒**：

| 数据项     | 说明                         | 数据格式示例         |
|-----------|------------------------------|---------------------|
| CPU 温度   | 当前 CPU 核心温度             | `65°C`              |
| CPU 占用率  | 当前 CPU 总体使用率           | `32%`               |
| GPU 温度   | 当前 GPU 核心温度             | `58°C`              |
| GPU 占用率  | 当前 GPU 总体使用率           | `45%`               |
| 内存占用率  | 当前物理内存使用率            | `68%`               |
| 网络上行速率 | 当前网络上行带宽              | `1.2 MB/s`          |
| 网络下行速率 | 当前网络下行带宽              | `5.6 MB/s`          |
| 当前时间    | 系统当前时间                  | `2025-01-15 14:30:00` |

### 2.2 蓝牙设备管理（底部区域）

#### 2.2.1 连接状态显示
- 底部区域显示当前蓝牙设备的连接状态
- 状态包括：`未连接` / `连接中` / `已连接` / `服务就绪`

#### 2.2.2 设备连接流程
- 未连接状态下，点击底部区域触发设备选择弹窗
- 弹窗内扫描并列出可用 BLE 设备
- 弹窗提供 **"只显示 DLG 开头的设备"** 勾选框（默认勾选），用于过滤目标设备
- 选择设备后自动发起连接

#### 2.2.3 目标硬件
- 主要连接目标：**DA14585** 蓝牙 SoC 设备
- 设备名称前缀：`DLG`

### 2.3 蓝牙数据同步

#### 2.3.1 同步机制
- 当蓝牙设备连接且服务就绪后，自动启动数据同步
- 同步周期：**每 1 秒** 一次
- 同步内容：2.1 节中所有监控数据

#### 2.3.2 通信协议
- 通信方式：BLE GATT
- 参考实现：`DlgClockBle` 项目的 [BleManager.kt](../../Coding/DlgClockBle/app/src/main/java/com/xmhzj/dlgclockble/BleManager.kt)

**GATT 服务与特征值：**

| 名称              | UUID                                     | 用途             |
|-------------------|------------------------------------------|-----------------|
| Service           | `0000ff00-0000-1000-8000-00805f9b34fb`   | 主服务           |
| Control Point     | `0000ff03-0000-1000-8000-00805f9b34fb`   | 控制指令写入      |
| ADC1 Value        | `0000ff02-0000-1000-8000-00805f9b34fb`   | 设备数据读取（通知）|
| Long Value        | `0000ff01-0000-1000-8000-00805f9b34fb`   | 时间同步/大数据读写 |

**已知协议命令：**

| 命令字节 | 功能           | 数据格式                                            |
|---------|----------------|---------------------------------------------------|
| `0x90`  | 切换时间显示格式 | 无附加数据                                          |
| `0x91`  | 同步时间        | 12 字节：年(2B LE)+月+日+时+分+秒+星期+农历年+农历月+农历日 |
| `0x92`  | 时间校准        | 3 字节：误差秒数(2B LE)                              |
| `0xA0`  | 固件升级-开始   | 136 字节：固件大小(2B LE)                             |
| `0xA1`  | 固件升级-数据   | 136 字节：固件数据块                                 |
| `0xA2`  | 固件升级-结束   | 结束标志                                             |

### 2.4 应用设置

#### 2.4.1 设置入口
- 主界面提供设置按钮（如齿轮图标），点击打开设置界面
- 设置界面可为独立弹窗或独立页面

#### 2.4.2 设置项

| 设置项           | 类型   | 默认值 | 说明                                                        |
|-----------------|--------|-------|-------------------------------------------------------------|
| 开机自启         | 开关   | 关闭  | 启用后应用随 Windows 系统开机自动启动                          |
| 启动后最小化到任务栏 | 开关   | 关闭  | 启用后应用启动时不显示主窗口，直接最小化到系统托盘               |

#### 2.4.3 设置持久化
- 所有设置项保存在本地配置文件中，应用重启后自动恢复
- 开机自启通过写入 Windows 注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` 实现

#### 2.4.4 退出行为
- 点击窗口关闭按钮时，弹出退出确认弹窗，提供两个选项：
  - **最小化到托盘**：窗口隐藏，应用继续在后台运行（系统托盘图标保留）
  - **退出应用**：彻底关闭应用进程
- 弹窗可勾选 **"不再询问，记住我的选择"**，勾选后后续关闭直接执行所选行为
- 该偏好可在设置界面中重置（恢复弹窗询问）

## 4. 非功能性需求

| 维度     | 要求                                          |
|---------|----------------------------------------------|
| 性能     | 监控数据采集与 UI 刷新不影响系统正常使用           |
| 兼容性   | 支持 Windows 桌面环境运行                       |
| 权限     | 需要蓝牙访问权限（BLE 扫描与连接）；开机自启需写入注册表 |
| 稳定性   | 蓝牙断连时自动回退到未连接状态，不崩溃             |
| 数据精度 | CPU/GPU 温度精确到 1°C，占用率精确到 1%，网速自动适配单位 |
| 持久化   | 用户设置与连接偏好本地持久化存储，重启不丢失        |
| 系统托盘 | 最小化到托盘后，托盘图标需支持双击恢复窗口、右键菜单退出 |
